ZeppelinOS runs on all EVM and eWASM-powered blockchains. It provides an on-chain set of upgradeable standard libraries, and an incentive structure to continually upgrade and patch itself.

Request services rather than re-implement them

Make use of an on-chain set of smart contract libraries for standard and common functions in your applications.

Continually upgrades and patches itself

ZeppelinOS creates a virtuous cycle where developers are incentivized to propose desirable upgrades, while users are ensured that the network is built and upgraded for their needs.

Upgrade the smart contracts on-chain

Fix security vulnerabilities or introduce new features to your apps without re-deploying new instances.

Ensure the code you are using stays secure and up-to-date

The ZEP token ensures that ZeppelinOS users receive the upgrades they need. Projects can stake tokens in favor of proposed Kernel upgrades.

The network then indicates developers the latest, most qualified version through the staking distribution.

Earn ZEP for auditing and developing updates to ZeppelinOS' libraries

Any developer can propose an upgrade to the ZeppelinOS kernel. Developers are compensated when ZEP holders stake their tokens in favor of the proposed upgrade.

The level of compensation is structured to ensure network upgrades are continually proposed, vetted, and adopted in a decentralized fashion.
